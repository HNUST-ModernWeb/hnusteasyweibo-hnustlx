# EasyWeibo 数据库设计文档
**项目**：简易微博社交平台  

---

## 一、表清单
1. 用户表（user）
2. 动态表（post）
3. 评论表（comment）
4. 点赞表（post_like）
5. 标签表（tag）
6. 动态标签关联表（post_tag）

---

## 二、表结构设计

### 1. 用户表 user
| 字段名 | 数据类型 | 约束 | 说明 |
| ---- | ---- | ---- | ---- |
| user_id | INT | PK、自增、非空 | 用户ID |
| username | VARCHAR(50) | UNIQUE、非空 | 用户名（不可重复） |
| password | VARCHAR(100) | 非空 | 密码 |
| avatar | VARCHAR(255) | 可空 | 头像路径/URL |
| register_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 注册时间 |
| user_type | TINYINT(1) | DEFAULT 0 | 0=普通用户，1=管理员 |
| is_deleted | TINYINT(1) | DEFAULT 0 | 逻辑删除 0=未删 |

**编码**：utf8mb4

---

### 2. 动态表 post
| 字段名 | 数据类型 | 约束 | 说明 |
| ---- | ---- | ---- | ---- |
| post_id | INT | PK、自增、非空 | 动态ID |
| user_id | INT | FK、非空 | 发布者ID，关联user |
| content | VARCHAR(200) | 非空 | 动态内容（支持表情） |
| post_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 发布时间 |
| comment_count | INT | DEFAULT 0 | 评论数 |
| like_count | INT | DEFAULT 0 | 点赞数 |
| is_deleted | TINYINT(1) | DEFAULT 0 | 逻辑删除 |

**编码**：utf8mb4

---

### 3. 评论表 comment
| 字段名 | 数据类型 | 约束 | 说明 |
| ---- | ---- | ---- | ---- |
| comment_id | INT | PK、自增、非空 | 评论ID |
| post_id | INT | FK、非空 | 所属动态ID |
| user_id | INT | FK、非空 | 评论者ID |
| content | VARCHAR(200) | 非空 | 评论内容（支持表情） |
| comment_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 评论时间 |
| is_deleted | TINYINT(1) | DEFAULT 0 | 逻辑删除 |

**编码**：utf8mb4

---

### 4. 点赞表 post_like
| 字段名 | 数据类型 | 约束 | 说明 |
| ---- | ---- | ---- | ---- |
| like_id | INT | PK、自增、非空 | 点赞ID |
| post_id | INT | FK、非空 | 被点赞动态ID |
| user_id | INT | FK、非空 | 点赞用户ID |
| like_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 点赞时间 |

**唯一约束**：UNIQUE (user_id, post_id) 防止重复点赞

---

### 5. 标签表 tag
| 字段名 | 数据类型 | 约束 | 说明 |
| ---- | ---- | ---- | ---- |
| tag_id | INT | PK、自增、非空 | 标签ID |
| tag_name | VARCHAR(30) | UNIQUE、非空 | 标签名（如#学习） |
| is_deleted | TINYINT(1) | DEFAULT 0 | 逻辑删除 |

**编码**：utf8mb4

---

### 6. 动态标签关联表 post_tag
| 字段名 | 数据类型 | 约束 | 说明 |
| ---- | ---- | ---- | ---- |
| pt_id | INT | PK、自增、非空 | 关联ID |
| post_id | INT | FK、非空 | 动态ID |
| tag_id | INT | FK、非空 | 标签ID |

**唯一约束**：UNIQUE (post_id, tag_id)

---

## 三、实体关系（ER 简要）
- 用户 1:N 动态
- 用户 1:N 评论
- 用户 1:N 点赞
- 动态 1:N 评论
- 动态 1:N 点赞
- 动态 N:M 标签（通过 post_tag 关联）

---

## 四、使用说明
- 所有表使用 **utf8mb4** 编码，支持 Emoji 表情
- 逻辑删除字段 `is_deleted` 避免物理删除数据
- 外键关联保证数据一致性
- 权限控制通过 `user_type` 字段实现
